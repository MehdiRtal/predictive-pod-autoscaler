    Architecture

    Predictive Pod Autoscaler (PPA) is designed as a modular, cloud-native system that integrates several industry-standard tools.

    Components

    1. Model CRD (managed by Kro)

    PPA uses Kro to define the Model Custom Resource. When you create a Model, Kro automatically orchestrates: - An Initial Training Workflow (Argo). - A Cron Workflow for hourly retraining (Argo). - A KServe InferenceService to serve the model.

    2. PPA Scaler

    A gRPC-based External Scaler for KEDA. It acts as the bridge between KEDA and the predictive models. - IsActive: Determines if the workload should be active (avoiding scale-to-zero if a load is predicted). - GetMetrics: Queries the KServe InferenceService for a prediction and returns the recommended replica count.

    3. PPA Pipeline

    A set of Python scripts packaged as a Docker image that: - Connects to Prometheus to fetch historical metrics. - Uses Prophet to train a time-series forecasting model. - Packages and uploads the model to S3-compatible storage.

    4. PPA Runtime

    A custom MLServer runtime designed specifically to load Prophet models and perform inference based on a "horizon" parameter (how far into the future to predict).

    5. Argo Workflows

    Orchestrates the training pipeline. It handles data fetching, model training, and updating the InferenceService.

    6. KServe

    Provides the infrastructure for serving the predictive models at scale, supporting features like scale-to-zero for the predictors themselves.
