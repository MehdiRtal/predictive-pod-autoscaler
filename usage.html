    Usage Guide

    Using PPA involves defining a Model for your workload and then configuring a KEDA ScaledObject to use that model.

    1. Define a Model

    Create a Model resource to tell PPA which workload to track and where to get the metrics.

    apiVersion: ppa.io/v1alpha1
kind: Model
metadata:
  name: my-app-cpu-prediction
  namespace: my-apps
spec:
  prometheusUrl: "http://prometheus-operated.monitoring:9090"
  queryType: "cpu" # or "memory"
  workload:
    type: "deployment"
    name: "my-app"

    What happens next? 1. Kro detects the new Model. 2. An Argo Workflow (my-app-cpu-prediction-init) is triggered immediately to train the first model. 3. A CronWorkflow is created to retrain the model every hour. 4. A KServe InferenceService named my-app-cpu-prediction is created to serve the model.

    2. Configure KEDA ScaledObject

    Now, create a KEDA ScaledObject that uses the PPA External Scaler.

    apiVersion: keda.sh/v1alpha1
kind: ScaledObject
metadata:
  name: my-app-scaler
  namespace: my-apps
spec:
  scaleTargetRef:
    name: my-app
  minReplicaCount: 1
  maxReplicaCount: 10
  triggers:
    - type: external-push
      metadata:
        scalerAddress: ppa-scaler.ppa:9090
        modelName: my-app-cpu-prediction
        horizon: "5"      # Predict 5 minutes into the future
        targetValue: "0.8" # Target 80% CPU usage

    3. Monitoring

    You can monitor the training progress by looking at the Argo Workflows:

    # Check workflow status
argo list -n my-apps

# Check model status and inference URL
kubectl get models my-app-cpu-prediction -o yaml
